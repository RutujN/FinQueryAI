How it works

The Hybrid RAG Pipeline Blueprint

We merge deep semantic vector coordinates with traditional keyword frequency matching to build an ultra-resilient search pipeline.

01

Ingestion & Chunking

Uploaded documents (PDF, TXT, MD) are cleaned, tokenized, and parsed into sliding window text segments (500-char overlap checks).

02

Dual Embedding Index

Chunks obtain deep 384-dim semantic embeddings via HuggingFace API and are cached inside SQLite vectors tables with metadata indexes.

03

Hybrid Ranker

Searches rank data by combining semantic cosine similarity scores and keyword overlap percentages to prevent model hallucinations.

04

Llama 3 Synthesis

Retrieved contexts are passed into Groq's high-speed Llama 3 API for structured, badge-referenced, and plotable answer synthesis.
